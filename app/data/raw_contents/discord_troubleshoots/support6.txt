So, I got it working, and to summarize how to monitor the status of two services (xandminer and xandminerd) on a remote Linux VPS and make the status available over the internet:

Add the following files in the folder /usr/local/bin:
A shell script (check_xandminer_status.sh) to check if the services are running.
A Python HTTP server (xandminer_status_server.py) to publish the response over the internet.

Add the following file to the folder /etc/systemd/system/:
A systemd service file (xandminer_server_status.service) to ensure the Python server and script starts at each system boot.

Set up a local running Uptime Kuma instance to check the server at http://<ipaddress>:8080/check_services every 60 seconds with an HTTP monitor type that has the "Accepted Status Codes" set to 200.

The file contents are generated by AI. I could provide them. 

Not sure if this setup is secure enough. Else you could use Cloudflare Zero Trust Tunneling prehaps, but maybe that is overkill.
