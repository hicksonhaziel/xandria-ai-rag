@DevNet :alert: :alert: :alert:  Attention pNode and vNode operators!! :alert: :alert: :alert: 

Make sure you are backing up your operator keypairs to a safe location. 
This key will be required to recover your pNode to a new server should you ever require to rebuild from scratch.
Currently only one pNode-keypair can be onboarded per pNode license purchased and there is no provision to use a new key to onboard again!
We are looking into a way to make this option available but have no timeline on an implementation.
Don't wipe or decommission a server without first backing up your pNode keypair.

For pNode operators, the key that is created during registration is located at:
/root/xandminerd/keypairs/pnode-keypair.json

You need to use a text editor (nano, vim, etc) to read this PRIVATE KEY and copy it to a safe location on a secure machine.

--

For vNode operators, the keys were created in your home directory if you followed the guide:

/home/sol/validator-keypair.json
/home/sol/vote-keypair.json
/home/sol/withdraw-keypair.json


Again, a text editor can be used to view the files and copy the contents to a secure machine.
Also, the withdraw keypair is required to withdraw funds from the vote account so the keys should not be stored together if there is economic value secured by the keys. (mainnet)

--

Be mindful that these are private keys, and if you lose them, there is no way to gain access to the contents or sign anything that was associated with them.
Also, if someone else gains access to these keys, they will have control of anything in or associated with those keys as well, so treat them securely!

--

To anyone running a pNode without a pNode license...there is one final step to get it up and running:
Run as root user:
fallocate /xandeum-pages -l 10g
ln -s /xandeum-pages /run/xandeum-pod

This creates a 10gb file, and symlinks it to where the pod software is looking for it allowing pod to start.
It will be added to docs with the next release.

Adjust the size to what you want to allocate and then check the service
This allows it to start to use the pRPC, but does not allow you to be part of the incentivized DevNet payouts.

sudo systemctl status pod.service
