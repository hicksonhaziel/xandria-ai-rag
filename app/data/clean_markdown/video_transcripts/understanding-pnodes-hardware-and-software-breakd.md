## Introduction to pNodes
The hardware requirements for pNodes are relatively low, with 2 gigs of RAM and 2 virtual CPUs being sufficient. There are no special CPU requirements, and 64 gigs of storage should be dedicated to the xandum network. The software requirements include installing the pNode software, which is currently available for Linux and Intel/AMD architecture.

## Software Deployment
The pNode software is easy to install and includes two components: the xan minor software and the pod software. The xan minor software is an admin interface that allows users to dedicate storage and view rewards, while the pod software communicates with the Atlas and is responsible for storing and retrieving data.

## pNode Network
The pNode network is currently in the devet phase, with a maximum of 300 pNodes and 3 pNodes per wallet. The network will eventually expand to include thousands of pNodes, with the goal of making it easy and secure for users to participate.

## Building on the zum Platform
The zum platform allows developers to build and test apps on the devet, with the goal of creating a decentralized storage layer. The platform includes a set of instructions, such as the Big Bang instruction, which creates a file system, and the Peak and Poke instructions, which allow users to read and write data.

## Merkel Proofs
Merkel proofs are a type of cryptographic proof that allows users to verify the integrity of data. They work by creating a tree of hashes, where each hash is a checksum of the data. This allows users to verify that the data has not been tampered with, without having to download the entire dataset.

## Data Security
The zum storage layer includes an optional encryption layer, which allows users to encrypt their data. The data is also chunked into pages, which are stored redundantly across multiple pNodes. This makes it difficult for bad actors to access or tamper with the data.
